Introduction

The Gene Ontology pipeline attempts to streamline the process of building a dataset for functional annotation. Based on the options selected on the dataset_form homepage, a large set of protein annotations will be extracted from the GOA database, filtered, and postprocessed. The resulting tab separated file can be used directly for training or evaluating models.


Benchmark Compatibility

Training and validation sets returned by our form are designed to avoid conflicts with all datasets from "Benchmarking Gene ontology Function Predictions Using Negative Annotations" (Vesztrocy, Dessimoz 2020). Datasets may also be made compliant with the CAFA Challenge by filtering by the proper maximum date.


Evidence Codes

The evidence codes in our form represent possible sources for protein - GO term associations. Select an evidence code to add all terms found using that method to your dataset.

A quick guide to evidence codes can be found here: Evidence Code Guide.


Propogate Annotations

Our Gene Ontology dataset maps proteins to the associated GO terms, but only actually stores the most specific term associated with a given protein. For instance, we might store an association between the protein Q13219 and the GO term "hexose biosynthetic process", but we wouldn't explicitely store an association between Q13219 and the more general term "metabolic process".

The propogate annotations option augments our dataset with these implicit associations before returning anything.


Max Date

Enter a maximum date to exclude recent protein - term associations from your dataset.


Negative Annotations (Coming Soon)

Very few negative annotations are explicitely provided by any dataset. However, there are methods for identifying very probably negative annotations.

Our pipeline extracts probably negative samples using the NegGOA process introduced in "NegGOA: Negative GO Annotations Selection Using Ontology Structure" (Fu, Wang, Yang, Yu 2020). A subset of these annotations, equal in size to the returned positive annotations, may be returned along with the other data.


Download Format

Your dataset will come in the form of a gzipped tar archive. On a bash terminal or command prompt, it can be unpacked with the command " tar -xvzf ". The resulting files correspond to the training, validation, and testing files of the Gene Ontology domains selected in the form.

Each file contains two tab-separated columns, with the first column storing UniProt protein IDs, and the second containing a comma separated list of the associated Gene Ontology terms.
